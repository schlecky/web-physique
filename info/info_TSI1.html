Architecture d'un ordinateurs¶

Un ordinateur est une machine lectronique qui execute un ensemble d'oprations arithmtiques sur des donnes. Les oprations sont dfinies dans le programme excut par l'ordinateur et les donnes sont stockes dans la mmoire de l'ordinateur.

Aujourd'hui un grand nombre d'appareils lectroniques sont (ou contiennent) des ordinateurs : ordinateurs, tlphones, tablettes, voitures, montres, ...

Un ordinateur contient gnralement les lments suivants :

  • Une source d'nergie (secteur, batterie) ;

  • Un microprocesseur qui est le composant qui execute les instructions et effectue les oprations ;

  • Une mmoire vive rapide qui n'est pas retenue lorsque l'alimentation est coupe (mmoire RAM);

  • Une mmoire de masse plus lente qui peut retenir l'information sans alimentation lectrique (disque dur, mmoire flash, ...) ;

  • Des priphriques d'entre sortie pour communiquer avec le monde extrieur ;

  • Des ports de communication pour connecter d'autres appareils lectroniques (autres ordinateurs).

Représentation des nombres¶

Dans un ordinateur, toutes les donnes (textes, images, sons, vidos, ...) sont stockes sous forme de nombres. L'lectronique et les mmoires des ordinateurs codent les nombres en base 2 (binaire) c'est dire que les seuls chiffres utiliss sont 0 et 1.

Représentation des nombres entiers¶

Dans la reprsentation habituelle en base 10 d'un nombre, chaque chiffre du nombre reprsente une puissance de 10. Par exemple $345 = 3\times10^2 + 4\times 10^2 + 5\times 10^0$.

En binaire c'est la mme chose sauf que chaque chiffre d'un nombre reprsente une puissance de 2. Par exempe le nombre binaire $1011_2 = 1\times 2^3 + 0\times 2^2 + 1\times 2^1 + 1\times 2^0 = 8+2+1 = 11_{10}$ (L'indice indique la base dans laquelle est crit le nombre).

Exercice Convertir les nombres binaires suivants en notation dcimale*: $11101$, $10101$, $1000$.

Pour convertir un nombre de la base 10 vers la base 2, il faut faire des divisions euclidiennes successives par 2 et noter les restes de droite gauche. Par exemple pour convertir $18_{10}$ en base 2 on effectue les divisions suivantes :

  • $18/2 = 9$ reste $0$
  • $9/2 = 4$ reste $1$
  • $4/2 = 2$ reste $0$
  • $2/2 = 1$ reste $0$
  • $1/2 = 0$ reste $1$

Et donc : $18_{10}$ = $10010_2$

Fichiers¶

Un programme informatique ncessite trs souvent d'enregistrer des donnes sur une mmoire non volatile, c'est dire qui persiste mme lorsque l'alimentation lectrique est coupe. Les donnes sur le disque dur d'un ordinateur sont stockes sous forme de fichiers.

On peut crer, lire et crire des fichiers partir d'un programme python.

Dans tous les cas, il faut ouvrir le fichier avec la fonction fich=open(nom,mode). nom est le nom de fichier ouvrir et mode est le mode d'ouverture du fichier. Les trois modes d'ouverture principaux sont :

  • 'r' : (read) pour ouvrir le fichier en mode lecture, on ne pourra rien crire dedans ;
  • 'w' : (write) pour ouvrir le fichier en mode criture. Si le fichier n'existe pas il est cr, si le fichier existe dj il est remplac ;
  • 'a' : (append) pour ajouter des donnes la fin d'un fichier. Si le fichier n'existe pas il est cr.

Pour lire les donnes d'un fichier on utilise la fonction fich.read(n) qui renvoie la chaine de caractre compose des n premiers caractres du fichier. Lorsuqu'on appelle nouveau la fonction read(n) ce sont les n caractres suivants qui sont renvoys. Si le nombre n est omis, la totalit du fichier est lue.

On peut lire une seule ligne de caractres en utilisant la fonction fich.readline().

Pour crire la chaine de caractres chaine dans un fichier on utilise la fonction fich.write(chaine).

Lorsqu'on a termin d'utiliser un fichier il faut le fermer en appelant la fonction fich.close().

Analyse d'un algorithme¶

Prouver qu'une boucle se termine effectivement¶

On a vu qu'une mauvaise utilisation d'une boucle while peut produire une boucle infinie, c'est dire un programme dont l'execution ne s'arrte jamais. On peut neanmoins prouver qu'une boucle se termine effectivement en utilisant un variant de boucle.

Définition :¶

Dans une boucle while o la condition d'arrt est la comparaison d'une variable et d'une constante de type entier positif, la variable est un variant de boucle si :

  • elle reste positive tout au long de l'algorithme
  • elle dcroit (ou croit) strictement chaque itration.

Dans ce cas on est sr que la boucle se termine aprs un nombre fini d'itrations.

Exemple :¶

On considre l'algorithme de la division euclidienne d'un nombre entier positif a par un autre nombre entier positif b :

In [49]:
def division(a,b):
    q=0 # Quotient
    r=a
    while(r>=b):
        q=q+1
        r=r-b
    return q,r

Dans cet algorithme, r est un variant de boucle car il est toujours positif et dcroit strictement chaque itration (b est ncessairement non nul)

Prouver qu'un algorithme donne le résultat attendu¶

L'utilisation d'un variant de boucle permet de prouver qu'un algorithme se termine effectivement. Mais il ne permet pas de s'assurer que le rsultat produit par l'algorithme est correct. Pour cela on utilise un invariant de boucle.

Définition¶

Un invariant de boucle est une proprit mathmatique qui :

  • est vraie avant d'entrer dans la boucle;
  • reste vraie aprs une itration
  • permet de monter avec la condition d'arrt de la boucle que le rsultat fourni est le rsultat attendu.

Dans une boucle for il n'y a pas de condition d'arrt explicite. On peut alors transformer la boucle for en une boucle while et appliquer ce que l'on a vu ci-dessus.

Exemple¶

Reprenons l'algorithme de division euclidienne de la partie prcdente. On peut montrer que $a=qb+r$ est un invariant de boucle. En effet :

  • Avant d'entrer dans la boucle on a $q=0$ et $r=a$ donc on a bien $a=qb+r$
  • En notant $q'$ et $r'$ les valeurs de $q$ et $r$ la fin de la boucle, on a $q'=q+1$ et $r'=r-b$ et $q'b+r'=(q+1)b+r-b = qb+r = a$. Donc si la proprit est vrai au dbut de la boucle elle reste vraie la fin de la boucle.
  • la fin de la boucle on a $r<b$ ce qui associ l'invariant de boucle assure que le rsultat est bien la division euclidienne de $a$ par $b$.

Temps d'exécution d'un algorithme (Complexité)¶

Définition¶

Un algorithme qui donne la bonne solution un problme peut n'avoir qu'un intrt limit si le temps qu'il met donner cette solution est trop long. Nous allons donc voir comment on peut estimer le temps d'execution d'un algorithme.

Plus prcisment, le temps d'execution d'un programme dpend du nombre d'instructions lmentaires effectues par le microprocesseur qui dpend trs souvent de la taille $n$ des donnes traiter. Bien videmment la vitesse de traitement des instructions par le processeur dpend de la machine sur laquelle le programme est excut. On ne pourra donc pas donner une valeur prcise du temps d'excution. Par contre, le temps d'excution $t(n)$ reste proportionnel au nombre d'instructions lmentaires $T(n)$ traiter qui dpend de la taille $n$ des donnes. Donc pour une machine donne il existe une constante $M$ qui dpend de la vitesse de la machine telle que :

$$t(n)=MT(n)$$

Le nombre d'instructions $T(n)$ est appel la complexit en temps de l'algorithme. En pratique on cherche dterminer comment $T(n)$ varie avec $n$, lorsque $n$ devient grand (comportement asymptotique).

  • Si $T(n)$ est indpendant de $n$, on note $T(n)=O(1)$. C'est une complexit en temps constant;
  • Si $T(n)$ est proportionnel $n$, on note $T(n)=O(n)$. C'est une complexit linaire;
  • Si $T(n)$ est proportionnel $n^2$, on note $T(n)=O(n^2)$. C'est une complexit quadratique;
  • Si $T(n)$ est proportionnel $k^n$, on note $T(n)=O(k^n)$. C'est une complexit exponentielle.

En pratique, pour dterminer la complexit d'un algorithme, on comptera souvent le nombre d'itrations d'une boucle. Mais a peut aussi tre plus compliqu.

On distingue plusieurs complexits en temps :

  • La complexit dans le pire des cas ;
  • la complexit dans le meilleur des cas ;
  • la complexit moyenne (la moyenne des complexits pour toutes les donnes d'entre possibles). Difficile dterminer.

On peut aussi valuer la complexit en mmoire qui donne une estimation de la quantit de mmoire utilise pour excuter un algorithme.

Dans le pire des cas, le caractre recherch n'est pas dans la chane et on doit effectuer la boucle $n$ fois ($n$ est la taille de la chane). Donc dans le pire des cas $T(n)=O(n)$.

Résolution d'équations¶

  • Dichotomie

Il existe des quations mathmatiques qui n'ont pas de solution mathmatique explicite, c'est dire que tant donne une quation de la forme $f(x)=0$, on ne peut pas exprimer la solution sous une forme simple $x=\ldots$. Il faut alors rsoudre numriquement l'quation, c'est dire trouver une valeur numrique approche de la solution.

Par exemple il est trs difficile (impossible) d'exprimer la solution de l'quation $\tan(x)=\sin(x^2)$ sous une forme simple.

Dans tout ce qui suit on cherchera rsoudre des quations de la forme $F(x)=0$. Par exemple l'quation prcdente est quivalente $F(x)=\tan(x)-\sin(x^2)=0$

Méthode de dichotomie¶

La mthode de la dichotomie consiste partir d'un encadrement d'une solution $x_{e}$ de l'quation rsoudre $x_e \in [a_1,b_1]$, $F(x_e)=0$ et de rduire progressivement l'encadrement.

L'encadrement initial ne contient qu'une solution et donc on a ncessairement $F(a_1)F(b_1)<0$. On calcule le point milieux de l'intervalle $[a_1,b_1]$ : $c_1=\frac{a_1+b_1}{2}$ et selon le signe de $c_1$ il remplace $a_1$ ou $b_1$ ($c_1$ prend la place de la borne du mme signe). On a alors un nouvel intervalle $[a_2,b_2]$ qui contient toujours $x_e$ mais de longueur deux fois plus petite.

On rpte cette procdure jusqu' encadrer la solution $x_e$ d'assez prs pour en avoir une approximation suffisante.

En python, on obtient la fonction suivante :

Résolution d'équations différentielles, méthode d'Euler¶

La mthode d'Euler est une mthode numrique de rsolution d'quations diffrentielles. Nous allons montrer le principe de la mthode et son application pour rsoudre numriquement une quation diffrentielle d'ordre 1.

Principe de la méthode¶

On souhaite rsoudre numriquement une quation diffrentielle de la forme $$\frac{dy}{dt}=f(y,t)$$ associe la condition initiale $y(0)=y_0$

Par exemple l'quation diffrentielle : $\frac{dy}{dt} = \cos(t)\sin(y)$, dans ce cas $f(y,t)=\cos(t)\sin(y)$. Ou plus simplement $\frac{dy}{dt}+4y=0$ o $f(y,t)=-4y$. L'quation diffrentielle est dfinie par la fonction $f(y,t)$ qui est une donne du problme.

On utilise un dveloppement l'ordre 1 de la fonction $y$ : $$y(t+dt)\simeq y(t) + y'(t)dt$$ soit $$y(t+dt)\simeq y(t)+f(y(t),t)dt$$

Donc connaissant la condition initiale $y(0)=y_0$ on peut calculer $y(dt)$ puis $y(2dt)$, etc.

L'approximation utilise revient identifier la courbe reprsentative de $y(t)$ sa tangente en tout point.

    t=[0]
    while(t[-1]<tmax):
        y.append(y[-1]+dt*f(y[-1],t[-1]))
        t.append(t[-1]+dt)
    return [t,y]

def pivot(A,k):
    """ Fonction qui dtermine le pivot dans la colonne k
    de la matrice A. Elle renvoit la ligne du premier 
    lment non nul de la colonne k """
    n=len(A)
    for i in range(k,n):
        if(A[i][k]!=0):
            return i
    print("Erreur, pas de pivot trouv")

def echange(A,i,j):
    """ change les lignes i et j de la matrice A """
    v=A[i]
    A[i]=A[j]
    A[j]=v

def transvection(A,i,j,lam):
    """ Modifie la matrice A en ajoutant  la ligne i 
    lam fois la ligne j """
    for k in range(len(A[0])):
        A[i][k]+=lam*A[j][k]
        
def resoudre(A,B):
    """ Rsout le systme linaire AX=B """
    n = len(A[0])
    # Matrice A sous forme chelonne
    for i in range(n-1):
        p = pivot(A,i)
        echange(A,p,i)
        echange(B,p,i)
        for k in range(i+1,n):
            transvection(A,k,p,-A[k][i]/A[i][i])
            transvection(B,k,p,-A[k][i]/A[i][i])
    
    # Matrice A sous forme chelonne rduite
    for i in range(1,n+1):
        B[n-i][0] = B[n-i][0]/A[n-i][n-i]
        A[n-i][n-i] = 1
        for k in range(n-i):
            transvection(B,k,n-i,-A[k][n-i]/A[n-i][n-i])
            transvection(A,k,n-i,-A[k][n-i]/A[n-i][n-i])
    return A,B

Intégration et dérivation numérique¶

Il arrive souvent que l'on souhaite obtenir numriquement la fonction drive ou l'intgrale (primitive) d'une fonction $f(x)$.

Le plus souvent c'est parce que la fonction $f(x)$ n'est connue que numriquement, c'est dire que l'on n'en connait qu'un nombre fini de valeurs ${f(x_1), f(x_2),\ldots, f(x_n)}$. Il peut galement arriver que la primitive $F(x)$ de la fonction $f(x)$ soit trs difficile calculer. (Pour un physicien, une primitive devient vite difficile calculer)

  • Drivation numrique
  • Intgration numrique

Dérivation numérique¶

La drivation numrique d'une fonction $f(x)$ repose sur la dfinition du nombre driv en $x_0$ :

$$ f'(x_0) = \lim_{h\rightarrow 0} \frac{f(x_0+h)-f(x_0)}{h} $$

On se contente alors de prendre une valeur de $h$ suffisamment petite et on crit :

$$ f'(x_0) \simeq \frac{f(x_0+h)-f(x_0)}{h} $$

On pourra donc utiliser la fonction Python suivante :

In [64]:
def derivee(f,x0):
    h=1e-4
    return (f(x0+h)-f(x0))/h

Testons cette mthode avec une fonction mathmatique dont la drive est connue : $$ f(x) = \cos(x) \quad f'(x) = -\sin(x) $$

Intégration numérique¶

L'intgration numrique est un peu plus complique que la drivation. On utilise le fait que l'intgrale $\int_a^b f(x) dx$ est gale l'aire sous la courbe reprsentative de $f(x)$ entre $a$ et $b$. Il faut donc calculer numriquement cette aire.

Méthode des rectangles¶

La mthode la plus simple est la mthode des rectangles qui consiste approximer la fonction par une srie de rectangles de largeur $p$ et de surface $S_i$:

Cette mthode est mise en oeuvre dans la fonction python suivante :

def integRectangles(f,a,b,n):
    """Intgre la fonction f entre a et b par la mthode des rectangles
    en utilisant n rectangles """
    p = (b-a)/n
    S=0
    for i in range(n):
        xi = a+p/2+i*p
        Si = f(xi)*p
        S += Si
    return S

Testons cette fonction pour calculer la primitive d'une fonction connue : $$ f(x) = \cos(x) \quad F(x) = \sin(x) $$

La manire utilise ci-dessus pour calculer la primitive de la fonction est trs peu efficace. Expliquer pourquoi et expliquer comment ou pourrait rendre ce calcul beaucoup plus efficace.

Sparer l'intervalle d'intgration en $n$ rectangles, avec $n$ fix a priori n'est pas forcment la mthode la plus efficace en moyenne. En effet pour une fonction qui varie lentement, le nombre de rectangles ncessaire sera plus faible que pour une fonction qui varie plus rapidement.

On pourra donc crire une nouvelle fonction qui adapte le nombre de rectangles jusqu' ce que le rsultat de l'intgrale ne varie plus :

In [68]:
def integRectanglesAdapt(f,a,b):
    diff=10
    n=10
    eps=1e-5
    integ = integRectangles(f,a,b,n)
    while(abs(diff)>eps):
        prec = integ
        n = 2*n
        integ = integRectangles(f,a,b,n)
        diff = integ-prec
    return integ
